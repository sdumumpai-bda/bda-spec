# standards/templates/ — Org-level template defaults

Templates ที่นี่คือ **org canonical** สำหรับทุก project ที่ใช้ bda-spec

โครงสร้าง template lookup:
1. `templates/<name>.md` — project override (ถ้ามี)
2. `standards/templates/<name>.md` — org default (fallback)

ถ้าทั้งสองที่มี → `templates/` ชนะ (project ปรับให้เข้ากับ context ตัวเองได้)

## ห้าม

- ห้ามแก้ไฟล์ใน `standards/templates/` โดยตรงในแต่ละ project — ใช้ `templates/` แทน
- การแก้ที่ standards ต้องทำผ่าน BDA AI Dev Standard repo + `/bda-sync`

(ดูใน `templates/` ของ project นี้ — ทั้งหมดเป็น org default ที่ลอกมา; เมื่อ project ต้อง override → copy ไฟล์เหล่านั้นเป็น `templates/<name>.md` แล้วแก้)

## หมายเหตุ

ใน clean install ของ bda-spec — `standards/templates/` มี file เดียวกับ `templates/` (เริ่มต้นเหมือนกัน). เมื่อ user แก้ที่ `templates/<name>.md` คือ override ของ project ตัวนั้น
