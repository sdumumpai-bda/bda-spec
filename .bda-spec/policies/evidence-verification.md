# Policy — Evidence Verification

## หลัก

ทุก claim ของผลลัพธ์ต้องมี evidence ที่ verify ได้

## ประเภท Evidence ที่ยอมรับ

| Claim | Evidence |
|---|---|
| Code change | git diff + commit hash |
| Test passed | test runner output + log file |
| Build success | build log + artifact location |
| Lint clean | lint output |
| Manual UAT | screenshot + tester name + date |
| Performance | benchmark output + environment |
| Security | scan tool output (sast/dast/sca) |
| Doc update | git diff of doc + before/after content |

## ที่ไม่ยอมรับ

- "Trust me" — no proof
- "Looks good" — no concrete check
- Screenshot ที่ไม่บอกว่าถ่ายเมื่อไหร่/ที่ไหน
- Test count ที่ไม่ match กับ output

## Storage

- Evidence ต้องเก็บใน vault: `docs/90-TestPlan/evidence/<YYYY-MM-DD>-<slug>/`
- มี manifest.json อธิบายแต่ละ artifact

## Retention

- Active project: เก็บ evidence ทั้งหมด
- Post-handoff: archive ตาม org retention policy
- ห้าม delete evidence ของ failed/regressed test
