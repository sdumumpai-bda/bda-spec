# Policy — Working Result

## หลัก

ส่งงานเมื่อ **ทำงานได้จริง** ในสภาพ "production-like" — ไม่ใช่แค่ compile ผ่าน

## ระดับ Working

| ระดับ | นิยาม | ใช้เมื่อ |
|---|---|---|
| **Compiles** | code build ไม่ error | minimum bar (ไม่พอสำหรับ handoff) |
| **Lint clean** | lint ไม่มี error | ก่อน commit |
| **Tests pass** | unit + integration tests pass | ก่อน PR |
| **Local smoke** | รัน flow หลักผ่านได้จริง local | ก่อน /bda-verify |
| **Staging smoke** | รันใน staging env ผ่าน | ก่อน production deploy |
| **Production verified** | ใช้งานจริงไม่มี regression | post-deploy |

## บังคับ

- `/bda-implement` ต้องไปถึงระดับ "Tests pass" minimum
- `/bda-verify` ต้องไปถึงระดับ "Local smoke" minimum
- Production deploy ต้องไปถึง "Staging smoke" minimum
- Handoff report ต้องระบุระดับที่ achieved

## ห้าม

- Claim "working" ถ้าแค่ compiles
- Skip levels โดยไม่ระบุเหตุผล
- Mark "done" ถ้า test ใหม่ที่เพิ่งเขียนไม่ run

## Exception

ถ้า block ที่ระดับใดระดับหนึ่ง (เช่น staging env ไม่พร้อม) → ระบุชัดเจน + เสนอ alternative verification
