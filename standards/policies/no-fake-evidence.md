# Policy — No Fake Evidence

## หลัก

ห้ามแต่ง evidence ทุกประเภท

## ขอบเขต

- Commit hash, branch name, PR number → ต้องมีจริงเท่านั้น
- Test count (pass/fail/skip) → ต้องมาจาก output ของการรันจริง
- Token count, AI cost → ต้องมาจาก tracking จริง หรือ mark "self-reported / not tracked"
- File content → ห้าม paraphrase แล้วอ้างเป็น quote
- URLs → ต้องเปิดได้จริง
- Screenshots → ห้ามสร้าง/แต่งภาพ
- Console/network logs → ต้องเป็น output จริง

## ถ้าทำไม่ได้

- เขียน `pending evidence` หรือ `not run — <reason>`
- ระบุ blocker เฉพาะที่ตรวจสอบได้
- เสนอทางเลือก verify

## บังคับตรวจ

`/bda-secure` + `/bda-verify` ตรวจ pattern เหล่านี้ และ block ถ้าเจอ

## Consequence

- Fake evidence ที่ถูกตรวจพบ → handoff invalid, ต้องทำใหม่
- Repeat offense → escalate ตาม policy ของ org
