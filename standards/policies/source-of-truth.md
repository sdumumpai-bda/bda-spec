# Policy — Source of Truth

## หลัก

ทุก fact ต้องมาจาก source-of-truth ที่ระบุ; ห้ามอ้าง memory ของ AI หรือ "ความรู้ทั่วไป"

## Hierarchy

1. **Vault** (`docs/`) — สำหรับ product/feature/function decisions
2. **Code** (source files) — สำหรับ current implementation
3. **`.bda-spec.yml`** — สำหรับ config
4. **`standards/`** — สำหรับ BDA process
5. **External docs** ที่อ้าง URL ชัดเจน — สำหรับ third-party

## ห้าม

- อ้าง requirement ที่ไม่อยู่ใน vault — ต้องสร้าง PRD/SRS/Function spec ก่อน
- อ้าง code behavior ที่ไม่ได้อ่าน source จริง
- อ้าง 3rd-party API ที่ไม่มี link/version

## ถ้า source-of-truth ขัดแย้ง

- Vault vs Code: เป็น drift — `/bda-doc --review` แล้วเลือก resolve direction (update doc หรือ update code)
- Standards vs Project: project follow standards เสมอ; ถ้าต้อง deviate ใช้ ADR

## วิธีอ้าง

ทุก claim ใน report:
- `[[<wikilink>]]` สำหรับ vault doc
- `path/to/file.ts:42` สำหรับ code
- URL + access-date สำหรับ external
