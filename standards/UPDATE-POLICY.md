# Update Policy — bda-spec standards

## Source of truth

Live standards: https://github.com/BigDataAgency/bda-ai-dev-standard

Local snapshot: `standards/` — pin version ใน `.bda-spec.yml` `standard.version`

## ห้าม

- ห้ามแก้ `standards/` ตรงในแต่ละ project
- ห้ามแก้ standard เพื่อ "ปรับให้เข้ากับ project" — ถ้าต้อง deviate ใช้ ADR + override ใน `templates/`

## วิธีปรับ standards

1. Feedback → BDA AI Dev Standard repo (issue / PR)
2. Standard repo merge → tag new version
3. `/bda-sync` ใน project → ดาวน์โหลด version ใหม่
4. Update `.bda-spec.yml` `standard.version` + `last_synced`
5. Log entry ใน `standards/SYNC-HISTORY.md`

## วิธี override (เมื่อ deviation จำเป็น)

1. สร้าง ADR ที่ `docs/70-Reference/ADR/ADR-NNNN-<reason>.md`
2. Override template ที่ `templates/<name>.md` (project-level)
3. Document ใน `CLAUDE.md` ว่า project ใช้ override อะไร
4. Submit feedback ไป standard repo ว่าทำไม override ควรเป็น default

## Breaking changes

- `MAJOR` bump (เช่น 0.x → 1.0): อ่าน CHANGELOG ละเอียด ก่อน sync
- `MINOR` bump: อ่าน CHANGELOG, review override impact
- `PATCH` bump: ปกติ safe

## Verification

หลัง `/bda-sync`:
- รัน `/bda-verify` กับ existing plan ที่ดี → ดูว่ายัง pass ไหม
- ถ้า fail → identify breaking change + fix override
